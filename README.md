# מערכת סידור עבודה – יחידה 6017

מערכת ווב לניהול סידורי עבודה עבור יחידה 6017. מאפשרת לאנשי צוות להגיש אילוצי זמינות, ולמנהל לייצר סידורי עבודה אוטומטיים תוך שמירה על אילוצים, תפקידים ומגבלות משמרות.

## סקירה כללית

| פרט | ערך |
|-----|-----|
| שפה | Node.js + Express |
| תבניות | EJS (RTL, עברית) |

- **`/`** – הגשת אילוצים (ציבורי)

- **`/admin`** – לוח בקרה מנהל (מוגן סיסמה)

### סכמת מסד הנתונים

```
roles            – תפקידים (שם, האם מתאים למפקד)
personnel        – אנשי צוות (שם, תפקיד, פעיל/לא)
facilities       – מתקנים
shifts           – משמרות (שם, שעות, סדר)
shift_requirements – כמה אנשים מכל תפקיד נדרשים בכל משמרת
constraints      – אילוצי זמינות של אנשי צוות (תאריך, משמרת, סיבה)
max_shifts_rules – מגבלת מקסימום משמרות לתקופה (לפי תפקיד/אדם)
schedule_periods – תקופות סידור (שם, טווח תאריכים)
schedule_assignments – שיבוצים בפועל (תקופה, תאריך, משמרת, מתקן, אדם, מפקד)
```

### אלגוריתם ייצור הסידור (`scheduleGenerator.js`)

האלגוריתם הוא **greedy עם איזון עומס עבודה**:

1. לכל יום בטווח → לכל מתקן → לכל משמרת:
2. לכל תפקיד נדרש, בונה רשימת אנשים זמינים:
   - **ראשית:** אנשים ללא אילוץ ביום/משמרת זה, ובתוך מגבלת המקסימום
   - **חלופה:** אנשים ללא אילוץ אך מעל המקסימום (רק אם הרשימה הראשית ריקה)
3. ממיין לפי כמות משמרות שצבר (מי עבד פחות – קודם)
4. משבץ את האנשים הנדרשים
5. בוחר מפקד: מבין השובצים שמוגדרים כ"ראויים למפקד", הלוקח הפחות שעות

**מפתח תקופה** (לספירת מגבלת משמרות):
- `week` → שנה-שבוע ISO (YYYY-WXX)
- `month` → שנה-חודש (YYYY-MM)
- `year` → שנה (YYYY)

### קידוד צבעים בסידור

| צבע | משמעות |
|-----|---------|
| 🟢 ירוק | שיבוץ תקין – ללא הפרות |
| 🟡 צהוב | האדם חרג ממגבלת המשמרות המקסימלית |
| 🔴 אדום | האדם שובץ בניגוד לאילוץ שהגיש |

בעת עריכה ידנית, המנהל יכול לשבץ מישהו גם אם הדבר מפר אילוץ – הסידור יסומן באדום אך יישמר.

### אבטחה

- סשן admin מוגן בסיסמה (מאוחסנת ב-DB, ניתנת לשינוי בהגדרות)
- `app.set('trust proxy', 1)` – נדרש ל-Heroku לתמיכה ב-HTTPS cookies
- משתני סביבה (DB, session secret) מוגדרים דרך `.env` / Heroku config vars

## הוראות שימוש – משתמשים

### הגשת אילוצים

1. פתח את האתר בדפדפן
2. בחר את שמך מהרשימה
3. לוח השנה יציג את הימים הקרובים
4. לחץ על יום שאינך יכול לעבוד:
   - **כל היום** – אילוץ על כל המשמרות
   - **משמרת ספציפית** – אילוץ רק על אותה משמרת
5. (אופציונלי) הזן סיבה בשדה "הערה"
6. לחץ **"שמור שינויים"**

כדי **לבטל** אילוץ – לחץ שוב על הריבוע הסגור ← יחזור לפתוח.

### צפייה בסידור

- סידורים שפורסמו יופיעו בקישור שתקבל מהמנהל
- הסידור מציג: תאריך ← מתקן ← משמרת ← אנשי הצוות השובצים
- מפקד המשמרת מסומן בכוכב ⭐

## הוראות שימוש – מנהל

### כניסה

1. לחץ **"כניסת מנהל"** בתפריט העליון
2. הזן את הסיסמה הנוכחית
3. לאחר כניסה מוצלחת תועבר ללוח הבקרה

### הגדרת מערכת (הכנה ראשונית)

**תפקידים** → הוסף תפקיד ← סמן "יכול לשמש כמפקד משמרת" לתפקידים מתאימים

**אנשי צוות** → הוסף את שמות אנשי הצוות ובחר תפקיד

**מתקנים** → הוסף מתקנים (מרכז, צפון וכד')

**משמרות** → הגדר שעות כניסה ויציאה, וסדר הצגה

**דרישות משמרת** (בלשונית "משמרות") → לכל משמרת, הגדר כמה אנשים מכל תפקיד נדרשים

### מגבלות משמרות

בלשונית **"מגבלות משמרות"**:
- בחר תפקיד (או אדם ספציפי)
- הגדר מקסימום משמרות
- בחר תקופה: שבוע / חודש / שנה

### ייצור סידור אוטומטי

1. לך ל**"סידור עבודה"**
2. תן שם לסידור
3. בחר תאריך התחלה וסיום
4. לחץ **"צור סידור אוטומטי"**

האלגוריתם ייצור שיבוצים תוך כיבוד אילוצים ומגבלות. שיבוצים חריגים יסומנו בצהוב/אדום.

### עריכה ידנית של הסידור

1. פתח סידור קיים
2. לחץ **"מצב עריכה"**
3. ניתן:
   - **להוסיף** אדם לתא (לחץ "+")
   - **להסיר** אדם (לחץ ✕)
   - **לשנות מפקד** (לחץ על הכוכב)
4. שיבוץ מנוגד לאילוץ יסומן **אדום** – מותר אך מסומן

### שינוי סיסמת מנהל

בלשונית **"הגדרות"** → שנה את הסיסמה

## התקנה ופריסה

### דרישות מקדמות
- Node.js ≥ 18
- מסד נתונים Azure SQL Server

### הגדרת `.env`

```env
PORT=3000
DB_SERVER=<your-server>.database.windows.net
DB_PORT=1433
DB_USER=<db-user>
DB_PASSWORD=<db-password>
DB_NAME=<db-name>
SESSION_SECRET=<random-secret-string>
ADMIN_PASSWORD=<admin-password>
```

### הרצה מקומית

```bash
npm install
node server.js
```

האפליקציה תיצור אוטומטית את טבלאות ה-DB בהרצה הראשונה.

### פריסה ל-Heroku

```bash
heroku create <app-name>
heroku config:set DB_SERVER=... DB_USER=... DB_PASSWORD=... DB_NAME=... SESSION_SECRET=... ADMIN_PASSWORD=...
git push heroku master
```

> **חשוב:** Heroku דורש `app.set('trust proxy', 1)` לתמיכה נכונה ב-session cookies מעל HTTPS. כבר מוגדר ב-`server.js`.
